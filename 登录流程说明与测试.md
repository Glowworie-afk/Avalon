# 微信登录流程（wx.login → code 换 openid）

这套流程的目的：安全地拿到玩家唯一身份 `openid`，全程不让 `appsecret` 离开服务器。

```
客户端                          你的后端                      微信服务器
  │                               │                              │
  │ 1. Taro.login() 拿临时 code   │                              │
  │ 2. POST /api/login {code} ──▶ │                              │
  │                               │ 3. code+appid+appsecret ──▶  │
  │                               │ ◀── 4. openid + session_key  │
  │                               │ 5. 用 openid 签发自己的 token │
  │ ◀────────── 6. { token } ──── │                              │
  │ 7. 之后每个请求带 token ────▶ │  authGuard 验 token → openid │
```

关键安全点：`appsecret` 只在第 3 步用到，永远只在后端。客户端拿到的是后端自己签的 token，跟微信的 session_key 无关。

## 新增 / 修改的文件

服务端（`server/`）：

- `config.ts` —— 统一读取环境变量（appid、secret、jwt 密钥、mock 开关）
- `utils/token.ts` —— 零依赖的 JWT（HS256）签发与校验，只用 Node 自带 crypto
- `middleware/auth.ts` —— `authGuard` 中间件，验 token 并把 openid 挂到 `req.openid`（第 7 步）
- `index.ts` —— Express 入口，挂载登录路由，含示例受保护接口 `/api/me`
- `http/login.ts` —— 已有，登录接口本身（mock 模式 + 正式模式）
- `.env` / `.env.example` —— 配置；`.env` 不要提交 git

客户端（`client/src/`）：

- `utils/request.ts` —— 请求封装，自动带 token、自动拼后端地址
- `utils/auth.ts` —— `login()` / `ensureLogin()`，封装第 1、2、4 步
- `app.ts` —— 启动时静默登录
- `pages/index/index.tsx` —— 演示页：登录按钮 + 调 `/api/me` 验证 token

## 怎么测（分三层，从易到难）

### 一、纯后端测（mock 模式，最快，不需要微信）

`.env` 里已设 `MOCK_LOGIN=true`，此时后端跳过微信，直接拿 `code` 当伪 openid。

```bash
cd server
npm install
npm run dev        # 启动，监听 http://localhost:3000
```

另开一个终端：

```bash
# 1) 健康检查
curl http://localhost:3000/health
# → {"ok":true,"mockLogin":true}

# 2) 登录，拿 token
curl -X POST http://localhost:3000/api/login \
  -H 'Content-Type: application/json' \
  -d '{"code":"test123"}'
# → {"token":"xxx.yyy.zzz","openid":"mock_test123"}

# 3) 用 token 调受保护接口（把上一步的 token 填进去）
curl http://localhost:3000/api/me \
  -H "Authorization: Bearer <把token粘这里>"
# → {"openid":"mock_test123"}

# 4) 不带 token —— 应返回 401
curl -i http://localhost:3000/api/me
```

预期结果（已实测通过）：第 3 步返回正确 openid；不带 token / 伪造 token 都返回 401；不带 code 登录返回 400。

### 二、客户端 + 后端联调（微信开发者工具，仍用 mock）

1. 后端保持 `npm run dev` 运行。
2. 用微信开发者工具打开 `client/` 目录，编译运行。
3. 重要：勾选 **详情 → 本地设置 → 不校验合法域名、TLS 版本以及 HTTPS 证书**，
   否则 `http://localhost:3000` 会被微信拦掉。
4. 首页点"登录"按钮，再点"调用 /api/me"，能看到 openid 即说明 token 链路通了。
   App 启动时其实已经自动登录了一次（见 `app.ts` 的 `ensureLogin`）。

此时 `code` 是真的（`Taro.login()` 拿的），但后端 mock 模式没拿去微信换，所以
openid 是 `mock_<code>`。这一步主要验证客户端 ↔ 后端的请求和 token 存取是否正确。

### 三、真实微信登录（关掉 mock）

1. 在微信公众平台拿到小程序的 AppID 和 AppSecret。
2. 改 `server/.env`：

   ```
   MOCK_LOGIN=false
   WX_APPID=你的appid
   WX_SECRET=你的appsecret
   JWT_SECRET=换成一段长随机串   # 生成：node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   ```

3. 客户端 `client/.env.development` 里填 `TARO_APP_ID="你的appid"`，并在微信开发者工具里用同一个 AppID 打开项目（匿名 AppID 是换不到 openid 的）。
4. 重启后端，重新走第二步的操作。这次后端会真的拿 code 去 `jscode2session` 换回真实 openid。

正式上线前还要：把后端部署到 https 域名，并在小程序后台
**开发 → 开发设置 → 服务器域名 → request 合法域名** 里加上该域名；
同时把 `client/src/utils/request.ts` 里的 `BASE_URL` 改成线上地址。

## 几个值得知道的点

- **token 里有什么**：header 和 payload 都是 base64，任何人都能解开看到 openid 和过期时间，
  所以别往 token 里塞密码。防伪靠的是 signature——没有 `JWT_SECRET` 就伪造不出合法签名。
- **token 多久过期**：默认 7 天（`TOKEN_TTL`）。过期后客户端 `request.ts` 会自动清掉 token，
  下次启动 `ensureLogin` 会重新登录。
- **session_key 去哪了**：本流程暂时没用它。它用于解密手机号、做数据签名等进阶场景，
  需要时建议存到 Redis（项目已装 ioredis），按 openid 做 key，别下发给客户端。
- **为什么没用 jsonwebtoken 库**：`utils/token.ts` 用 Node 自带 crypto 实现了标准 HS256 JWT，
  零依赖、可与任何 JWT 库互通。想换成 `jsonwebtoken` 库也可以，接口保持 `signToken`/`verifyToken` 即可。
